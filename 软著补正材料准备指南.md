## 一、新增/强化功能完整说明

### 1.1 系统概述

智课 Agent（全称：智课Agent——一款融合向量匹配技术的智能复习辅助系统 V1.0）是基于 **RAG（Retrieval-Augmented Generation，检索增强生成）** 架构的多课程 AI 助教系统。系统将课程教材文档通过嵌入模型转为向量存入 ChromaDB 向量数据库，用户提问时先从向量库检索相关教材片段，再将片段作为上下文发送给大语言模型（豆包 API）生成基于教材内容的回答。

核心创新点：**不依赖 AI 模型的训练数据，而是以用户上传的课程教材为唯一知识来源**，通过"向量检索 + AI 生成"实现准确、可控的课程问答。

### 1.2 系统功能模块

#### 模块一：用户认证与安全管理

| 功能 | 说明 |
|------|------|
| 用户注册 | 用户名、邮箱、密码、安全问题（可选） |
| 用户登录 | JWT Token 认证，登录后获取访问令牌 |
| 密码重置 | 通过邮箱验证或安全问题验证重置密码 |
| 权限分级 | 普通用户 / 管理员 两级权限 |
| 会话保持 | Token 存储在浏览器 localStorage，30 分钟有效期 |
| 密码安全 | 采用 pbkdf2_sha256 哈希存储，不可逆 |

#### 模块二：课程选择与导航

| 功能 | 说明 |
|------|------|
| 课程卡片展示 | 6 门课程以卡片式布局展示，含课程图标、描述、对话统计 |
| 课程快速进入 | 点击卡片进入对应课程的 AI 对话界面 |
| 历史记录入口 | 顶部导航栏直达对话历史页面 |

#### 模块三：AI 智能问答（核心功能）

| 功能 | 说明 |
|------|------|
| 多课程支持 | 6 门计算机科学课程：软件工程、操作系统、计算机网络、数据结构与算法、数据库系统、编译原理 |
| 查询优化 | 每门课程配置专属查询优化规则，将用户的自然语言问题优化为更精准的检索查询 |
| 向量检索 | 从课程 ChromaDB 向量数据库检索 Top 5 相关教材片段 |
| RAG 增强生成 | 将检索到的教材片段构造为 Prompt 上下文，调用豆包大模型生成基于教材内容的回答 |
| 三层索引策略 | 教材按"概念增强层→段落上下文层→细粒度分块层"三级索引，提高检索召回率 |
| 快速问题建议 | 对话界面提供预设问题按钮，方便用户快速提问 |
| 对话历史保存 | 每次问答自动保存到数据库，支持按会话查看历史 |
| Fallback 容错 | AI 服务不可用时返回友好提示，不会报错崩溃 |

#### 模块四：对话历史管理

| 功能 | 说明 |
|------|------|
| 历史列表 | 按时间倒序展示所有对话会话 |
| 课程筛选 | 按课程类型过滤历史记录 |
| 关键词搜索 | 搜索对话标题和内容 |
| 分页展示 | 历史记录过多时自动分页 |
| 继续对话 | 从历史记录直接进入之前的对话上下文 |
| 删除会话 | 支持删除不需要的对话记录 |

#### 模块五：管理员知识库控制台（本次新增）

| 页面 | 功能 |
|------|------|
| 系统概览 | 6 门课程的整体状态一览：教材是否上传、向量库是否构建、AI 是否加载。运行状态面板展示 API 状态、AI 服务状态、模型名称和 API Key 配置状态 |
| 课程知识库 | 教材上传（拖拽式 DOCX 文件上传）→ 向量知识库构建（后台异步任务，实时查看日志）→ 构建完成后自动热重载（无需重启后端）。每门课显示教材状态、向量库文档块数、AI 加载状态，提供上传/构建/重载/检索测试按钮 |
| 检索测试 | 选择课程 → 输入测试查询 → 查看"原始查询→优化后查询"的转换结果 → 查看向量检索返回的教材片段及其 metadata（来源、类型、优先级等） |
| 系统状态 | 查看 API 状态、数据库连接状态、AI 服务状态、当前使用的模型名称、嵌入模型名称、API Key 配置状态、已加载课程列表。提供"重新加载 AI 服务"按钮 |

### 2.4 关于 AI/检索效果不理想的应对

如果某些测试查询未能检索到理想的教材片段，或 AI 回答不够准确：

1. **查询太模糊** → 使用更精确的术语。例如用"瀑布模型"而非"那个做软件的模型"。
2. **教材未覆盖该知识点** → 检查原始 DOCX 教材中是否包含该内容。向量检索只能找到教材中存在的知识。
3. **查询优化规则未覆盖** → 在 `course_config.py` 的 `query_optimizations` 中添加新的优化规则。
4. **截图方案**：优先使用已验证有效的查询词（如"瀑布模型""软件生命周期""进程调度"等），这些查询在现有教材中都有良好覆盖。

**推荐的"安全"测试查询（能在现有教材中找到答案）**：

| 课程 | 查询词 | 预期效果 |
|------|--------|----------|
| 软件工程 | "瀑布模型" | 检索到瀑布模型的定义、特点 |
| 软件工程 | "软件生命周期" | 触发查询优化，检索到各阶段描述 |
| 软件工程 | "需求分析" | 检索到需求分析相关内容 |
| 操作系统 | "进程调度" | 检索到进程调度算法 |
